# سامانه ثبت آگهی خرید و فروش دست دوم

این پروژه، یک سامانه جامع کلاینت-سرور برای ثبت و مدیریت آگهی‌های خرید و فروش کالای دست دوم است که به عنوان پروژه پایانی درس **برنامه‌سازی پیشرفته** در دانشگاه صنعتی امیرکبیر توسعه داده شده است.

## 📝 معرفی پروژه
در این سامانه، کاربران می‌توانند حساب کاربری بسازند، آگهی‌های خود را همراه با تصویر و مشخصات ثبت کنند، در میان آگهی‌های دیگران جستجو و فیلتر کنند، با فروشندگان وارد گفت‌وگو (چت) شوند و آگهی‌های مورد علاقه خود را نشان کنند. 

این سامانه دارای یک **پنل مدیریت پیشرفته** است که مدیر سیستم می‌تواند آگهی‌های جدید را بررسی (تایید/رد) کرده و آمارهای کلیدی سیستم را مشاهده کند. 

برای ایجاد تجربه کاربری بهتر، سیستم **امتیازدهی به فروشندگان**، **آپلود و نمایش چندتایی عکس‌ها (اسلایدر)**، **به‌روزرسانی خودکار (Real-time) چت‌ها** و **مدیریت هوشمند تصاویر پیش‌فرض** نیز در این سامانه پیاده‌سازی شده است. معماری این سیستم به صورت Client-Server بوده و ارتباطات از طریق REST API انجام می‌شود.

---

## 🛠 تکنولوژی‌های استفاده شده

* **زبان برنامه‌نویسی:** Java 17
* **Backend:** Spring Boot (Spring Web, Spring Data JPA)
* **Frontend:** JavaFX و FXML
* **پایگاه داده:** SQLite
* **احراز هویت و امنیت:** JWT (JSON Web Token)
* **مدیریت وابستگی‌ها:** Maven

## 🏃 نحوه اجرای پروژه

این پروژه از دو بخش کاملاً مجزا (فرانت‌اند و بک‌اند) تشکیل شده است. برای اجرای صحیح، ابتدا باید بک‌اند و سپس فرانت‌اند را اجرا کنید.

### ۱. تنظیمات پایگاه داده
این پروژه از پایگاه داده سبک `SQLite` استفاده می‌کند؛ بنابراین نیازی به نصب سرور دیتابیس خارجی نیست. با اولین اجرای بک‌اند، فایل پایگاه داده (`ap_project.db`) به صورت خودکار ساخته شده و تمامی داده‌ها به صورت کاملاً پایدار ذخیره خواهند شد.

### ۲. اجرای Backend
یک ترمینال باز کرده و وارد پوشه `backend` شوید و پروژه را با دستور زیر اجرا کنید:
> cd backend
> mvn spring-boot:run

*(سرور روی پورت `8080` اجرا شده و آماده دریافت درخواست‌ها خواهد بود).*

### ۳. اجرای Frontend
یک ترمینال جدید باز کرده، وارد پوشه `frontend` شوید و برنامه را با دستور زیر اجرا کنید:
> cd frontend
> mvn javafx:run

*(پس از اجرا، پنجره گرافیکی ورود به سامانه باز خواهد شد).*

## 📸 گزارش قابلیت‌های پیاده‌سازی شده (تصاویر محیط برنامه)

### ۱. ثبت‌نام و ورود کاربران (احراز هویت با JWT)
کاربران می‌توانند در سیستم ثبت‌نام کرده و با استفاده از نام کاربری و رمز عبور وارد شوند.
**(اینجا باید یک اسکرین‌شات از صفحه فرم ورود یا ثبت‌نام برنامه بگذارید)**

### ۲. داشبورد اصلی، جستجو و فیلتر آگهی‌ها
لیست آگهی‌های فعال نمایش داده می‌شود. کاربران می‌توانند آگهی‌ها را فیلتر کنند. (آگهی‌های فاقد عکس به صورت هوشمند با تصویر پیش‌فرض نمایش داده می‌شوند).
**(اینجا باید یک اسکرین‌شات از داشبورد اصلی برنامه بگذارید)**

### ۳. مشاهده جزئیات آگهی، اسلایدر عکس و امتیازدهی
نمایش اطلاعات کامل آگهی. در این صفحه قابلیت **اسلایدر عکس‌ها** برای آگهی‌های دارای چند تصویر وجود دارد. همچنین کاربران می‌توانند به فروشنده **امتیاز (۱ تا ۵)** بدهند.
**(اینجا باید یک اسکرین‌شات از صفحه جزئیات آگهی که دکمه‌های قبلی/بعدی عکس و بخش امتیازدهی در آن مشخص است بگذارید)**

### ۴. ثبت و مدیریت آگهی‌ها (آپلود همزمان چند تصویر)
کاربران می‌توانند آگهی جدید ثبت کنند و در یک مرحله **چندین عکس** را برای یک آگهی آپلود کنند.
**(اینجا باید یک اسکرین‌شات از صفحه فرم ثبت آگهی جدید بگذارید)**

### ۵. سیستم گفت‌وگوی زنده (Real-time Chat)
سیستم پیام‌رسانی داخلی برای تبادل پیام بین خریدار و فروشنده. این سیستم بدون نیاز به رفرش دستی، پیام‌های جدید را به صورت خودکار دریافت و نمایش می‌دهد.
**(اینجا باید یک اسکرین‌شات از صفحه چت برنامه بگذارید)**

### ۶. علاقه‌مندی‌ها (نشان‌شده‌ها)
لیست آگهی‌های ذخیره‌شده برای دسترسی سریع‌تر.
**(اینجا باید یک اسکرین‌شات از صفحه آگهی‌های نشان‌شده بگذارید)**

### ۷. پنل مدیریت و آمار سیستم
مدیر سیستم می‌تواند علاوه بر بررسی آگهی‌های در انتظار، **آمارهای کلی سامانه** (تعداد کاربران، کل آگهی‌ها، آگهی‌های معلق) را به صورت گرافیکی مشاهده کند.
**(اینجا باید یک اسکرین‌شات از صفحه آمارهای پنل ادمین بگذارید)**

## 📊 گزارش مشارکت اعضای گروه

* **زینب رفیع زاده:** 
مسئولیت اصلی من در این پروژه، مدل‌سازی دامنه، طراحی پایگاه داده و توسعه کامل بخش Backend بود. من ساختار دیتابیس را با استفاده از JPA/Hibernate پیاده‌سازی کردم و ارتباطات موجودیت‌ها را با رعایت اصول شی‌گرایی تنظیم نمودم. سیستم احراز هویت و امنیت (Security) را با استفاده از توکن‌های JWT توسعه دادم. در نهایت، پیاده‌سازی تمامی REST APIها (شامل سیستم چت، آمار داشبورد ادمین، سیستم امتیازدهی و مدیریت چندتایی فایل‌ها)، مدیریت خطاها و آزمون منطق سمت سرور بر عهده من بود.

* **آرمیتا فرحناک:** 
مسئولیت اصلی من طراحی و توسعه کامل بخش Frontend و ایجاد یک رابط کاربری تعاملی بود. من با استفاده از JavaFX، صفحات مختلف سامانه را طراحی کردم. سپس با پیاده‌سازی لایه API Client، کلاینت را به سرور متصل کرده و ارسال و دریافت داده‌ها را مدیریت کردم. همچنین پیاده‌سازی قابلیت‌های پیشرفته ظاهری (نظیر اسلایدر تصاویر، آپدیت خودکار پیام‌ها در چت، مدیریت تصاویر پیش‌فرض و پنجره انتخاب همزمان چند فایل) و تست یکپارچگی سیستم توسط من انجام شد.
